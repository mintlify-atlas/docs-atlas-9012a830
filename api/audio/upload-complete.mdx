## POST /api/v1/fileUploaded/:audioId

Notifies the system that an audio file has been successfully uploaded to S3. This triggers the audio processing pipeline which extracts metadata and converts the audio to standard formats.

### Authentication

<ParamField header="Cookie" type="string" required>
  Must include `access_token` cookie with valid JWT token.
</ParamField>

### Path Parameters

<ParamField path="audioId" type="string" required>
  MongoDB ObjectId of the audio metadata entry (returned from presigned URL generation)
</ParamField>

### Request Body

<ParamField body="key" type="string" required>
  S3 object key where the file was uploaded (returned from presigned URL generation).
  Will be automatically trimmed.
</ParamField>

### Response

<ResponseField name="success" type="boolean">
  Indicates whether the upload confirmation was successful
</ResponseField>

<ResponseField name="data" type="object">
  Updated audio metadata
  
  <ResponseField name="status" type="string">
    Always "processing" after successful confirmation
  </ResponseField>
  
  <ResponseField name="id" type="string">
    MongoDB ObjectId of the audio entry
  </ResponseField>
  
  <ResponseField name="fileName" type="string">
    Name of the uploaded file
  </ResponseField>
</ResponseField>

<ResponseField name="message" type="string">
  Status code on success
</ResponseField>

<ResponseField name="error" type="null">
  Always null on success
</ResponseField>

### Status Codes

- **202 Accepted** - Upload confirmed, processing started
- **400 Bad Request** - Invalid audioId or key
- **403 Forbidden** - Authentication required or user not authorized

### Example Request

```bash
curl -X POST https://api.example.com/api/v1/fileUploaded/507f1f77bcf86cd799439011 \
  -H "Content-Type: application/json" \
  -b cookies.txt \
  -d '{
    "key": "507f191e810c19729de860ea-a1b2c3d4-my-song.mp3"
  }'
```

### Example Response

```json
{
  "success": true,
  "data": {
    "status": "processing",
    "id": "507f1f77bcf86cd799439011",
    "fileName": "my-song.mp3"
  },
  "message": "202",
  "error": null
}
```

### Processing Pipeline

After calling this endpoint, the following background processing occurs:

1. **Status Update**: Audio status changed from "uploading" to "processing"
2. **Job Queued**: Background job added to Bull queue for processing
3. **File Download**: Audio file downloaded from S3 to worker
4. **Metadata Extraction**: Audio properties extracted using FFmpeg
5. **Format Conversion**: Audio converted to MP3 (if needed)
6. **Upload Processed Files**: Converted audio uploaded back to S3
7. **Database Update**: Metadata updated with processing results
8. **Status Update**: Status changed to "ready" or "failed"

<Note>
  Only the user who created the audio entry can confirm its upload. The status is automatically set to "processing" and a background job is triggered to extract metadata and process the audio.
</Note>

<Warning>
  Call this endpoint only after successfully uploading the file to S3 using the presigned URL. Calling it prematurely may cause processing errors.
</Warning>
